What is this?

This is a demonstration initramfs image, with every kernel option turned on and a basic inittab that starts up shells on the first five virtual consoles. You would use this image to make sure the image will boot on your target hardware, and to poke around and see how things work :)

Note that most of the memory usage is because of all of the kernel modules stored in /lib/modules, which in turn takes up space in the initramfs filesystem (ramdisk): 

~ # uname -a
Linux lack 2.6.31-lack #1 SMP PREEMPT Sat Aug 1 11:52:48 PDT 2009 i686 GNU/Linux

~ # du -sh /lib/modules
61.7M   /lib/modules
You can save a lot of space after booting by deleting the modules that you don't need from the initramfs filesystem.

What's available?

  • vmlinuz-X.X.XX and initramfs-lack-X.X.XX.DATE.cpio.gz, the kernel and initramfs image; can be used to PXEboot or with SYSLINUX for USB flash drives
  • lack.VERSION.DATE.zip - a zipfile containing the kernel and initramfs images
  • lack.VERSION.DATE.tar - a tarball containing the kernel and initramfs images
  • lack.VERSION.iso - an ISO image you burn with CD writing software

Download the tarball or the zipfile or the ISO image or the vmlinuz and initramfs-blank-mbr files, or download them all!

Usage

Burn the ISO, create a bootable USB flash drive with syslinux or PXEboot the kernel and initramfs image, then boot up the system.

Enjoy!
